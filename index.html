¿Qué es ser Full Stack Developer?

Un Full-stack developer es un programador con un perfil que abarca conocimientos y maestrías desde la parte back-end (core de un proyecto) hasta lo referente al front-end (vistas y presentación del producto). Un programador de estas características se maneja bien con las bases de datos, con REST APIs y con lenguajes de presentación y manipulación del DOM como Javascript. En la actualidad es un perfil cada vez más demandado por las empresas, ya que un técnico de estas características es una pieza fundamental para el buen devenir de un proyecto informático.

¿Por qué Full Stack?

Un Full-stack developer necesita manejar con soltura, o al menos tener conocimientos y experiencia, en un amplio abanico de tecnologías y lenguajes de programación como los que aquí se enumeran:

  1. HTML5 y CSS3: Toda aplicación orientada a web tiene su esqueleto basado en Html y CSS. Html genera la estructura y el contenido del sitio web y CSS se encarga de enriquecer visualmente ese contenido.

  2. Javascript: El lenguaje Javascript es el rey de Internet en estos momentos. Es imposible manejar y entender todos los frameworks y librerías que cada día son lanzados a Internet y que se basan en las bonanzas de Javascript. Javascript es el único lenguaje que funciona de forma nativa en el navegador y por ello es el amo del mundo Front. Si manejas JS y te asomas a NodeJs puedes llegar a ser un programador muy serio y cotizado.

  3. Lenguajes Backend (Java, Python o PHP): Un lenguaje Backend se encarga de la lógica de negocio y de las operaciones más complejas sobre los datos de la aplicación a desarrollar. Hay decenas de lenguajes orientados al backend, pero los más importantes y sólidos son un par: Java, C# y Python. Como serio aspirante tenemos a Node, pero aún falta mucho para que las grandes compañías apuesten por él.

  4. Bases de Datos: Se debe poseer un buen entendimiento de los modelos de base de datos que existen en el mercado, aunque en la actualidad se dividan en 2: bases de datos relacionales y no relacionales. Es importante comprender las bonanzas de bases de datos relacionales como las SQL, pero sin desdeñar las NoSQL como MongoDB.
